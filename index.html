Rust 1.90.0 released

Ignoring concurrency (as discussed in the other subthread), Rust provides universally better aliasing analysis than C++, and better-in-practice than C. This gives nearly every idiomatic non-trivial Rust program a small but measurable speed advantage over the equivalent idiomatic C or C++.

To be more concrete about this: In Rust, safe references are mutable-xor-shared. That is, mutable references are never allowed to alias (except via reborrowing), and immutable references must not point at bytes that may change out from under them (unless protected by UnsafeCell). In safe Rust, this is enforced by the borrow checker, but even in unsafe Rust, this is considered an aliasing restriction and violating it is UB. Of course, unsafe Rust can use raw pointers, which have no aliasing restrictions, but that is not idiomatic. Similarly, UnsafeCell is mostly used to implement synchronization primitives, and occasionally as a single-threaded escape hatch to the borrow checker (see the std::cell module).

Because this rule is stricter than C and C++ are with their pointers, Rust can directly emit noalias annotations on nearly all safe references, and so it has roughly the same performance advantage as Fortran. In C, you can theoretically emulate this with the restrict keyword, but I don't think that is done very often in practice, because the language does not provide the necessary invariants to pervasively enforce it throughout a large codebase. At most, you'll see it on some function parameters here and there.
